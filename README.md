# sv-monitor-discord (Discord Server Resource Monitor Bot)

Discord のスラッシュコマンドを利用して、Bot が動いているホストサーバーのリソース状態（CPU、メモリ、ディスク使用量、直近1秒のネットワーク通信速度）をリアルタイムに取得・表示する Python 製の Discord Bot です。

Docker コンテナ化されており、環境変数と Docker Compose を使用して簡単にデプロイできます。

---

## 🚀 主な機能

- **`/status` スラッシュコマンド**: 
  - 実行された瞬間のサーバーリソース情報を測定し、Discord に送信します。
  - **管理者限定権限**: デフォルトで管理者（Administrator）のみが実行できるように制限されています。
  - **プライベート返信 (Ephemeral)**: コマンドを実行した本人にのみ結果が表示されるため、チャンネルの他のメンバーにサーバー情報を露出させません。

---

## 🛠️ セットアップと起動手順

### 1. Discord Bot の作成とトークンの取得
1. [Discord Developer Portal](https://discord.com/developers/applications) にアクセスします。
2. **New Application** を作成し、Bot を追加します。
3. **Bot** タブから **Token** を生成（Reset Token）し、手元に控えておきます。
4. **Privileged Gateway Intents** はデフォルトのままで問題ありません（メッセージ本文の読み取り等は不要です）。
5. **OAuth2** -> **URL Generator** から以下のスコープを選択し、生成された URL から Bot をサーバーに招待します。
   - `bot`
   - `applications.commands` (スラッシュコマンド同期用)

### 2. 環境変数の設定
リポジトリルートにある `.env.sample` を `.env` にコピーし、取得した Discord Bot トークンを設定します。

```bash
cp .env.sample .env
```

`.env` ファイルをエディタで開き、以下のようにトークンを設定します。

```env
DISCORD_BOT_TOKEN=ここに取得したBotトークンを貼り付けます
```

### 3. Docker Compose による起動
Docker がインストールされている環境で、以下のコマンドを実行してコンテナをビルド・起動します。

```bash
docker compose up -d --build
```

起動後、Bot が Discord 上でオンラインになり、管理者権限を持つユーザーが `/status` スラッシュコマンドを使用できるようになります。

## ⚠️ セキュリティに関する重要な注意点（パブリック公開用の精査情報）

本リポジトリを安全に運用するために、以下の点について十分にご理解の上ご利用ください。

### 1. ホストディレクトリのマウントリスク (`docker-compose.yml`)
- `docker-compose.yml` 内で、ホストマシンのディスク使用量を正確に監視するために、ホストのルートディレクトリをコンテナ内の `/host` にリードオンリーでマウントしています (`/:/host:ro`)。
- **リスク**: もしコンテナ自体や Python スクリプト、依存ライブラリに脆弱性があり、コンテナが乗っ取られた場合、**ホストマシンの全ファイルがリードオンリーで覗き見られるリスク**があります（`.ssh` 鍵や機密設定ファイルなど）。
- **対策案**: 
  - 本 Bot は、ご自身が管理する信頼できるプライベートなサーバー上でのみ稼働させてください。
  - ディスク監視が不要、または特定のディレクトリのみの監視で十分な場合は、`docker-compose.yml` の `volumes` 設定からマウントを削除、あるいはマウント範囲を制限（例: `/var/lib/docker:/host/var/lib/docker:ro` などに変更）し、それに合わせてプログラム側のパスも調整してください。
  - Docker を使わずに、ホスト上で直接 Python スクリプトをデーモンとして実行することでも、この Docker マウントによるリスクを回避できます。

### 2. ネットワーク監視の仕様について
- コンテナ内のネットワークインターフェースを監視するため、現在の設定では「コンテナ自身の通信速度」が測定されます。
- ホストマシン全体の通信速度を厳密に測定したい場合は、`docker-compose.yml` で `network_mode: "host"` を指定する必要がありますが、これはホストのネットワーク名前空間をコンテナに共有させるため、さらなるセキュリティ上のトレードオフが発生します。

### 3. コマンド実行権限の管理
- `monitor_bot.py` 内で、`/status` コマンドには `@app_commands.default_permissions(administrator=True)` を設定し、管理者以外がコマンドを実行できないようにしています。
- パブリックな Discord サーバーで運用する場合は、意図しない一般ユーザーにサーバー情報が漏洩しないよう、この権限設定を改変しないようにしてください。
